    6. Saved runs (replay & copy)

    Runs live under agent/runs/. Pick a saved run — its steps load below immediately (no browser, no execution). Then use Open interactive replay for step-by-step control in one Chromium tab. If a step fails, that tab stays open so you can fix the app or edit the graph and run again. Stop any live recording in section 1 before opening interactive.

      Interactive replay (step control — default)

      Opens one browser for this session. Steps are numbered in the list below; Run marks ✓ or ✗. Use ＋ on a row to splice a new step (form opens in the same block). Force fix is deterministic: it probes the interactive tab for your existing selectors (visible or attached, same as running the step). If none match you get a clear "not on this page" message. If one matches, it reorders primary/fallback locator text. For upload steps only, when [data-testid="upload-area"] is attached on that tab, it may also append canonical upload fallbacks before reordering. It does not call an LLM, invent selectors from error text, or log in / navigate for you. Save graph to disk persists stepgraph.json. A failed step does not close the browser.

      Suggest fix (LLM) would be a separate, explicit control (cost, latency, safety policy) — it is not implemented in this build. Do not expect semantic repair from Force fix; use site-side test ids / unique file inputs (see PORTING_NOTES.md upload contract) plus Pick or manual graph edits when needed.
